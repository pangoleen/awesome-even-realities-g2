# Contribution Guidelines

Thanks for helping make this list better. Please follow these rules so the list stays curated and passes [awesome-lint](https://github.com/sindresorhus/awesome-lint).

## Adding to the list

- Make an individual pull request for each suggestion.
- Add your entry to the bottom of the most appropriate category. If no category fits, propose a new one in your pull request.
- Use the format: `- [Name](link) - Description.`
- Keep descriptions concise, in the present tense, starting with a capital letter and ending with a period.
- Use the project's real name as the link title.
- New categories or improvements to existing categorization are welcome.
- Check your spelling and grammar.

## Quality standards

To be included, a project should be genuinely useful to someone building on the Even Realities G2. The list is a curation of the best, not an index of everything.

- The project must be relevant to G2 development specifically (not only G1).
- Prefer projects that are maintained, documented, and working.
- Do not add unmaintained, archived, deprecated, or undocumented projects. If something has clear historical value but is no longer maintained, note that in the description.
- No self-promotion of low-effort or placeholder repositories.

## Before you submit

- Confirm the link works and points to the canonical repository or page.
- Run `npx awesome-lint` against your fork and fix any reported issues.
- Confirm your addition is not already present, and is not a duplicate of an existing entry.

Pull requests that do not follow these guidelines may be asked to revise before merging.
